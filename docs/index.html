A cyclic inequality on the simplex
Given non-negative real numbers a, b, c with a + b + c = 1, prove that the cyclic sum a²b + b²c + c²a is at most 4/27.

Continuous evolution

Every problem makes the system stronger

This isn't a static tool. Every proof attempt — success or failure — generates training signal. The agent accumulates knowledge, strategies, and mathematical intuition over time.

01

Lemma Bank compounds

Proved sub-lemmas are stored and reused across problems. A lemma from Problem A becomes the key to solving Problem B — mathematical knowledge transfers and accumulates.

02

Winning strategies remembered

"Cyclic inequality → decompose + nlinarith with sq_nonneg" is remembered. The next cyclic inequality starts from this winning playbook instead of from scratch.

03

Self-improving training data

Every (theorem, proof, verified?) triple becomes fine-tuning data. Success vs failure pairs form preference data. The underlying model continuously improves from its own experience.

04

Smarter resource allocation

The system learns which problems need Light mode (minutes) vs Heavy mode (hours). Easy problems get minimal compute. Hard problems get proportionally more — like a skilled mathematician managing their time.
