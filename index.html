PUTNAM 2019 · 题目 A2
单纯形上的轮换不等式
给定非负实数 a, b, c 且 a + b + c = 1，证明轮换和 a²b + b²c + c²a ≤ 4/27。

步骤 1 · 领域识别 + 证明规划
ON_PROBLEM_START 钩子分类领域，匹配策略插件
在写下任何代码之前，智能体先构建高层证明计划 —— 如同数学家在白板上勾勒思路。
证明规划器分析目标结构，将高层数学直觉转化为可执行的战术序列。是智能体的"数学大脑"。 假设生成器自动猜测可能需要的辅助假设与关键引理，为证明搭建脚手架。 模式库存储历史上成功的证明模式（如"轮换不等式→Schur 变体"），供规划器快速匹配。
策略选定: "轮换不等式 → Schur 变体" 关键观察: 等号成立于 (a,b,c) = (2/3, 1/3, 0) 及其轮换 所需辅助引理: • h₁: a·b ≤ (a+b)²/4 (AM-GM 乘积界) • h₂: 推广的 Schur 界用于轮换和
智能体不盲目尝试策略 —— 它先对数学结构进行推理，再生成任何代码。

步骤 2 · 知识检索
从 100,000+ 引理库中搜索构建模块
混合检索（关键词 + 语义向量）遍历 Mathlib 数学库，再由 LLM 重排结果。
BM25 稀疏检索基于 TF-IDF 的传统检索，精确匹配引理名称和策略关键词，速度极快。 语义嵌入将定理语句编码为向量，捕捉深层语义相似性，找到名称不同但本质相关的引理。 LLM 重排器用大模型理解当前目标语境，将最相关的引理排到前面，过滤噪声。 策略建议器根据目标类型自动建议 nlinarith, polyrith, ring_nf 等策略，节省搜索时间。
检索到的顶级前提: 1. sq_nonneg — ∀ x, 0 ≤ x² (辅助项的关键) 2. mul_self_nonneg — ∀ x, 0 ≤ x * x 3. div_le_div_right — 单调除法 4. Finset.sum_le_sum — 逐点 → 求和界 5. mul_le_mul_left — 标量乘法 建议策略: nlinarith, polyrith, ring_nf, positivity

步骤 5 · 错误诊断
理解为什么失败，而不仅是知道失败了
错误诊断器对 Lean 报错进行语义分类（策略失败/类型不匹配/超时/sorry残留），选择针对性修复。 根因分析器不仅看表面报错，还追溯到证明策略层面的根本原因，避免重复犯同样的错误。
根因: nlinarith 无法直接处理三变量轮换表达式。 多项式搜索空间对自动算术引擎来说太大了。 修复策略: 分解为更小的、可独立证明的子引理。

步骤 6 · 经验提取
即使失败的证明也产出可复用知识
3 条中间引理虽然来自失败的证明，但各自独立通过了 Lean 内核验证，被存入引理库。
引理提取器从失败的证明中提取所有独立成立的中间结论，化失败为知识积累。 引理验证器对每条提取的引理进行独立的 Lean 内核验证，确保引理库的绝对可靠。 引理库跨问题的知识库。问题 A 的引理可能成为解决问题 B 的关键 —— 数学知识的迁移与积累。
没有任何尝试是浪费的。每次失败都在引理库中沉淀验证过的知识，加速后续的证明。
