Architecture

Two-layer agent design

Universal agent infrastructure decoupled from theorem-proving pipeline. Each directory is a capability. Each file is a responsibility.

agent/brain

Claude LLM interface with multi-role prompting — proof generator, critic, repair specialist, decomposer

agent/memory

Working memory for current problem state; episodic memory for cross-problem learning

agent/strategy

Meta-controller: Light → Medium → Heavy mode escalation with reflection and confidence estimation

agent/tools

Tool registry + executor: Lean automation (exact?, simp, omega) and CAS bridge (SageMath)

prover/sketch

Proof plan generation + hypothesis proposal before any code is written

prover/codegen

Sorry-based scaffolding: generate skeleton, then close each sorry via REPL + automation

prover/verifier

Lean 4 kernel verification + interactive REPL + goal extraction + integrity checking

prover/repair

Error diagnosis → root cause analysis → targeted fix generation → patch application

prover/lemma_bank

Cross-rollout experience sharing: extract proved sub-lemmas from failed proofs, reuse in next round

prover/decompose

Complex theorem → sub-goal tree → solve independently → compose final proof

benchmarks

7 datasets: miniF2F, PutnamBench, FATE-M/H/X, ProofNet, FormalMATH + builtin

knowledge

Mathlib RAG: BM25 + embedding hybrid retrieval over 100K+ declarations

SFT Data

Every (theorem, proof, verified) triple becomes supervised fine-tuning data. The model learns what "correct" looks like.

DPO Pairs

Failed vs successful proofs for the same theorem form preference pairs. The model learns to avoid common mistakes.

Winning strategies are remembered. "Cyclic inequality → decompose + nlinarith with sq_nonneg hints" is stored for future similar problems.
